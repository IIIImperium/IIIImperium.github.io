title: pwndbg基础

  1. 基本用法 (启动 GDB 与 pwndbg)

安装完成后，你只需像往常一样启动 GDB，pwndbg 就会自动加载并改变 GDB 的外观和行为。

启动 GDB：

调试可执行文件：

gdb <可执行文件路径>

例如：gdb ./a.out

附加到正在运行的进程：

gdb -p <进程ID>

启动后的界面：
一旦 GDB 启动，pwndbg 会立即接管并显示一个全新的界面，通常包括：

Code (代码区): 当前程序计数器 (PC/RIP/EIP) 附近的汇编代码。

Registers (寄存器区): 所有 CPU 寄存器的当前值。

Stack (栈区): 当前栈帧中的数据，包括函数参数、局部变量和返回地址。

Legend (图例): 颜色和符号的解释。

  1. pwndbg 常用命令和功能

pwndbg 增强了 GDB 的许多功能，并添加了大量自己的命令。你仍然可以使用所有的标准 GDB 命令。

核心 GDB 命令（pwndbg 会美化其输出）

r / run：运行程序。

start：在 main 函数处设置断点并运行，在 main 函数入口停止。

b / break ：设置断点（例如：b main, b *0x400500, b function_name, b file.c:line_number）。

c / continue：继续执行直到下一个断点或程序结束。

n / next：执行下一行源代码（步过函数调用）。

ni / nexti：执行下一条汇编指令（步过函数调用）。

s / step：执行下一行源代码（步入函数调用）。

si / stepi：执行下一条汇编指令（步入函数调用）。

info b / info breakpoints：查看所有断点信息。

x /

：查看内存内容。

N (数量): 要显示的单位数量。

F (格式): x (十六进制), d (十进制), i (指令), s (字符串), c (字符)。

U (单位大小): b (字节), h (半字, 2字节), w (字, 4字节), g (巨字, 8字节)。

示例：x /20wx $rsp (查看栈顶 20 个 4 字节的十六进制值)。

pwndbg 特有或增强的命令

以下是 pwndbg 提供的一些最有用的命令：

  1. 上下文和显示控制

context：手动刷新并显示所有上下文信息（寄存器、栈、代码等）。

telescope

/ tele
：以美观的格式递归地查看指针链所指向的内存内容，是 pwndbg 的明星命令之一。

例如：tele $rsp (查看栈顶内容及其指向)。

例如：tele

(查看指定地址开始的 count 个 QWORD 的内容)。

hexdump

[length]：以十六进制和 ASCII 形式显示内存内容。

例如：hexdump $rsp 0x40。

vmmap：显示内存映射（segments），类似于 cat /proc//maps。

elfheader：显示当前模块的 ELF 头信息。

main_arena：显示 malloc 的 main_arena 结构。

  1. 搜索和查找

search / find ：在所有可读内存区域中搜索指定的值（字符串、整数、指针）。非常强大，可以快速定位内存中的数据。

search “my_string”

search 0xdeadbeef

search &my_variable (搜索变量的地址)

search_gdb_history：搜索 GDB 历史命令。

  1. 栈分析

stack / st：显示当前栈帧的内容，通常与 context 中的栈视图一致。

retaddr：尝试定位并显示当前函数的返回地址。

canary：尝试找到并显示栈溢出保护（Canary）的值。

  1. 堆分析

heap：显示堆块（chunks）的摘要信息。

heap chunks：详细显示所有堆块的头部和数据。

heap bins：显示各种 malloc 的 bin (tcache, fastbin, smallbin, largebin) 的内容。

arena：显示 malloc_state 结构体（main_arena 或其他 arena）的详细信息。

  1. 实用工具

rop / ropgadget：在当前加载的模块中搜索 ROP gadget。

ropgadget –depth (指定搜索深度)。

ropgadget –chain “; “ (尝试构建 ROP 链)。

shellcode：提供了生成各种 shellcode 的功能（如 shellcode i386/linux/execve）。

checksec：显示可执行文件的安全属性（ASLR, NX, Canary, PIE, RELRO）。

disass / disassemble：反汇编函数或代码区域。

dumpargs：显示当前函数栈上的参数。

sym / symbol ：查找符号的地址。

cyclic / cyclic_find：用于生成和查找 Metasploit 风格的 Cyclic pattern，辅助确定栈溢出偏移。

cyclic 100 (生成一个 100 字节的模式)。

cyclic_find 0x61616161 (查找覆盖 EIP/RIP 的模式值，确定偏移)。

patch：在内存中打补丁（修改字节）。

patch byte

patch word

patch dword

patch qword

patch string

“new_string”

  1. 其他

dump ：将内存区域写入文件。

load

：将文件内容加载到内存。

highlight

：高亮显示某个地址的内存区域。

reset_context：重置并刷新上下文显示。

config：查看和修改 pwndbg 的配置选项。

  1. pwndbg 常用工作流程示例

假设你正在分析一个存在栈溢出的 CTF 二进制文件 pwnme。

启动调试：

gdb ./pwnme

pwndbg 会显示初始上下文。

查看安全属性：

checksec

确认 ASLR, NX, Canary, PIE 等防护。

设置断点并运行：

b *main+ # 或 b vuln_function 等
r

程序在断点处停止。

查看栈和寄存器：
pwndbg 自动显示。你可以滚动查看 stack 区域，或者使用 telescope $rsp 深入查看。

构造溢出数据 (如果发现栈溢出)：

生成 Cyclic pattern：cyclic 200

复制输出，并将其作为输入提供给程序：run < (echo “output_from_cyclic”)

程序崩溃后，pwndbg 会显示寄存器状态，特别是 RIP/EIP 的值。

查找偏移：cyclic_find <RIP/EIP的值>，例如 cyclic_find 0x61616161。这将告诉你从输入开始到覆盖返回地址的字节数。

构造 ROP 链 (如果需要绕过 NX)：

rop 或 ropgadget 命令搜索 gadgets。

利用 search 命令查找 system 函数、/bin/sh 字符串等关键地址。

print system (GDB 命令) 可以直接获取函数地址。

内存写入和修改：

使用 patch 命令在调试时临时修改内存内容，例如 patch byte 0x400000 0x90。

使用 set $reg = 修改寄存器值。

查看内存映射：
vmmap 帮助你了解程序加载的模块和内存区域。

  1. 配置 pwndbg

pwndbg 有许多可配置的选项，可以通过 config 命令查看和修改。

config：显示所有配置选项及其当前值。

config ：显示特定选项的详细信息。

set ：修改选项的值。

例如：set context-sections code stack reg (只显示代码、栈和寄存器)。

例如：set enable-colors off (关闭颜色)。

你也可以将常用的配置写入你的 ~/.gdbinit 文件中，例如：

source ~/pwndbg/gdbinit.py
set context-sections code stack reg
set show-retaddr-hints off
